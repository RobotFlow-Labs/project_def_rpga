# Task PRD-0102: Define Data and Asset Schemas

> PRD: PRD-01 | Depends: PRD-0101
> Files: 4 | Lines: ~240 | Time: ~75min

## What to Build
- Add typed models for `CaptureRecord`, `ViewSpec`, `BenchmarkProtocol`, and `AssetManifest`.
- Encode the paper’s weather / distance / pitch / azimuth protocol in a reusable schema.

## Acceptance Criteria
- [ ] Schemas validate and round-trip JSON.
- [ ] Protocol builder emits 2 weathers, 4 distances, 5 pitch values, and 36 azimuth bins.
- [ ] Tests cover invalid metadata and missing asset roots.

## Implementation Notes
- Keep the schemas filesystem-agnostic so later tasks can source data from CARLA, rosbag, or manual captures.
- Prefer explicit units in field names, e.g. `distance_m`, `pitch_deg`.
